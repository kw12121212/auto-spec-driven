## Resolved

- [x] Q: Should `spec-driven-init` create roadmap assets by default?
  Context: This determines whether roadmap is a first-class repository asset or
  an optional add-on.
  A: Yes. `init` should scaffold roadmap assets from the first version.

- [x] Q: Should milestones contain only approved changes, or both candidate
  ideas and approved changes?
  Context: This determines whether roadmap can capture future options without
  prematurely turning them into execution work.
  A: Milestones should contain both, with candidate ideas and planned changes
  kept distinct.

- [x] Q: Can milestones be marked complete manually?
  Context: This determines whether roadmap status is advisory or tied to actual
  archive state.
  A: No. A milestone is complete only when all listed planned changes are
  archived.
